⚠️ VULNERABILITY EXPLANATION

This site used client-side hidden form fields to pass the price in the POST body. The backend blindly trusted the price parameter without server-side recalculation.


What you did with Burp Suite:

  • Intercepted the POST https://httpbin.org/post request
  • Found the hidden field: price=XXXX in the request body
  • Modified it to: price=0
  • Forwarded the tampered request → Server accepted it!

🛡️ HOW TO FIX THIS (Remediation)

  • ✅ Never trust client-side data — Always recalculate price server-side using product ID + quantity from your DB
  • ✅ Store cart in server session — Keep cart state on the server, not in hidden fields
  • ✅ HMAC signing — If passing values client-side, sign with HMAC-SHA256 and verify server-side
  • ✅ Re-validate at payment — Before charging, re-fetch prices from DB and compare
  • ✅ Anomaly detection — Flag orders where price < expected threshold for manual review
